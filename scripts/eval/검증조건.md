# 리포트 검증 조건 (기준 정의서)

이 문서는 리포트 검증의 **정답지와 채점 기준**을 정의한다.
실행 방법(렌더→생성→평가 메커니즘)은 [README.md](./README.md), 회차별 결과는
각 회차 폴더의 `검증파일.md`. **이 문서는 "무엇을 좋은 리포트로 보는가"의 source of truth다.**

---

## 0. 대원칙 — 리포트 종류마다 정답이 다르다

가장 흔한 실수: **모든 리포트를 한 기준으로 채점하는 것.** 특히
"사주가 TCI(기질검사) 결과와 일치하는가"로 사주를 평가하면 안 된다 —
사주는 TCI를 예측하는 도구가 아니고, 둘은 다른 것을 측정한다. 둘이
다른 인물을 그리는 것은 결함이 아니라 **융합 리포트가 다룰 재료**다.

모든 리포트는 **효용 3축 + 경험 3축 + 위생**으로 채점한다(§1, §1-B). 종류별로
"무엇을 정답으로 보는가"만 아래처럼 달라진다.

| 리포트 | 무엇이 "정답"인가 | 종류별 유의점 |
|---|---|---|
| **사주(personal-saju)** | 그 사주로 사용자가 얻는 **효용** | 명리 정확도/TCI일치 ❌ |
| **기질(tci-report)** | 입력한 7차원 점수의 **충실한 반영** | 점수 충실성 추가 확인 |
| **융합(tci-saju-fusion)** | 사주·기질의 **공명/긴장을 정직하게 통합** | 자기이해 축 비중 큼 |
| **가족(family-saju)** | 구성원 간 **관계 효용** | 명리 생극 정합 추가 확인 |

> 공통 6축(고민해소·자기이해·미래행동 / 새로움·재미·쉬움) + 위생. "정답"은
> 종류마다 다르지만 **잣대(6축)는 같다** — 회차 간·종류 간 비교를 위해.

## 1. 효용 3축 (사용자가 정한 핵심 기준)

사주·융합·가족·기질 공통으로 보는 "사용자에게 쓸모 있는가"의 3축.

### 축1 — 고민 해소
> 생애주기 고민을 다루는 카테고리/풀이를 제공하는가.

- 사용자의 연령·생애단계에 맞는 고민들(진로·이직, 돈·재정, 연애·결혼,
  가족, 건강, 인간관계, 자기계발, 번아웃, 미래 불안)을 **폭넓게** 다루고,
  각 카테고리에 **막연한 위로가 아닌 실질 단서/방향**을 주는가.
- 평가 시: 리포트가 다루는 고민 카테고리를 목록으로 추출 → 빠진 고민·
  얄팍하게 스친 고민을 지적.

### 축2 — 자기 이해
> "내가 이런 사람이구나"를 또렷이 알게 하는가.

- 자기상이 **구체·일관·입체적(강점과 그림자, 모순까지)**이면 자기이해가 생긴다.
  **막연·뻔함·누구에게나 맞는 말(바넘)**이면 아무것도 못 배운다.
- 정답(실제 성격)이 없으므로 "구체성·일관성·입체성"으로 측정한다.
- 사주↔기질의 인물상 차이는 감점이 아니다 — **통합해 더 깊이 이해할
  재료인가, 그냥 혼란인가**로 판정(융합이 봉합하면 통합).

### 축3 — 미래 + 행동
> 다가올 변화를 (기대감 있게) 알려주고, 지금 할 일을 주는가.

- 미래: 흐름·변화를 알려주되 시기가 **구체적**("2026년 하반기" ⃝ / "언젠가" ✗),
  **공포가 아니라 기대감·준비감**.
- 행동: "지금/이번 분기 할 일"이 **실행 가능·측정 가능**하게 떨어지는가.
- 미래와 행동이 **연결**되는가(이 변화가 오니 → 그래서 지금 이걸).

## 1-B. 경험 3축 (읽는 경험의 질)

효용(쓸모)과 별개로, "이 글을 끝까지 즐겁게 읽고 또 보고 싶은가"의 축.
쓸모만 있고 이게 없으면 "맞는 말이지만 안 읽힌다"가 된다.

### 축4 — 새로움 (사주 많이 본 사람도 새롭게 느끼는가)
> 사주를 여러 번 본 경험자에게도 신선한가.

- 기준 독자는 **사주를 이미 많이 본 사람**. "일간이 ○○라 다정하다",
  "물 일간은 지혜롭다" 같은 사주 책·유튜브의 **정형화된 통설**을 그대로
  반복하면 낮다. 같은 데이터라도 **신선한 프레이밍·의외의 연결·구체적 적용**으로
  "오 이건 처음 듣는 관점인데" 하게 만드는가.
- **바넘과 구분**: 바넘 = *누구에게나* 맞는 말(자기이해 축에서 잡힘).
  새로움 결핍 = *사주 좀 본 사람이면 다* 들어본 말. 둘 다 진부함이지만 다른 축.
- 측정: "사주 경험자 입장에서 처음 듣는/관점이 신선한 대목"과 "이미 닳도록
  들은 통설" 대목을 가른다.

### 축5 — 재미 (글 자체가 재밌는가)
> 정보가 맞는 것을 넘어, 읽는 재미가 있는가.

- 위생의 "가독성"(지루하지 않음 = 감점 방지)을 **넘어서는 가점 축**. 비유가
  신선하고 위트·리듬이 있고, 다음 줄이 궁금하고, **캡처해서 공유하고 싶은
  문장**이 있는가. 읽다가 미소·감탄·"헉 소름"이 나는 지점이 있는가.
- ↔ 정보는 정확한데 교과서처럼 건조하거나, 좋은 말인데 밋밋하면 낮다.
- 측정: "공유하고 싶은 문장" 추출 / 미소·감탄 지점 / 지루해서 건너뛴 구간.

### 축6 — 쉬움 (어려운 용어 없이 술술 읽히는가)
> 일반인이 사전 없이 막힘 없이 읽어 내려가는가.

- 한자·명리 용어(일간·월지·대운·오행·화생토 등), 현학적 한자어, 난해한
  추상어 **없이** 누구나 한 번에 이해되는가. 문장이 길고 꼬여 다시 읽게
  만들지 않는가. 술술 미끄러지듯 읽히는가.
- ↔ 한자 병기, 풀이 없는 명리 용어, 만연체가 흐름을 한 번이라도 끊으면 낮다.
- **위생의 "한자 노출"과의 관계**: 위생은 *룰 위반 여부*(한자가 있나)를 감점으로
  본다. 축6은 *독자 경험*(용어·문장·명료성을 합쳐 실제로 술술 읽히는가)을 가점으로
  본다. 둘은 같은 문제의 두 얼굴이니 함께 움직이는 게 정상이다.
- 측정: 읽다가 멈칫하게 한 단어·문장을 짚는다 / 중학생이 읽어도 이해되는가.

### 위생 (감점 요인)
> 효용·경험이 좋아도 이게 깨지면 신뢰가 무너진다.

- **내적 모순**: 같은 사람/사주를 앞뒤 다르게 서술(예: 같은 시기를 입력기↔출력기로).
- **운명 단정·공포 조장**: "~할 것이다" 단정, 불안 조장.
- **반복·물타기**: 같은 비유·처방·통찰의 반복, 잘라도 손실 0인 만연체.
  ★길이 자체는 감점 대상이 아니다 — 유료 프리미엄 리포트는 풍성해야 한다.★ 감점하는 것은
  *반복·물타기*(같은 말을 변주만 해서 늘린 것)이지 길이가 아니다. 길어도 매 문단이 새로운
  내용·서로 다른 장면이면 만점. 짧아서 얄팍한 것이 길어서 풍성한 것보다 나쁘다.
- **버그**: 미치환 `{{변수}}`, `2026년 2026-06월` 류 날짜 오류, 프롬프트 지시문 노출.
- **한자/명리 용어 노출**: 일반인 비노출 원칙(CLAUDE.md) 위반.
- **톤**: 현재 제품 톤과 불일치. AI 리포트/상담은 반말 팩폭 톤을 유지하되,
  순수 UI 문구처럼 보이는 설명은 과도하게 무례하지 않아야 한다. 가족 리포트에서
  가족 구성원을 비난하거나 낮춰 부르면 신뢰 위반.

## 2. 점수 앵커 (10점 만점, 축마다 공통)

점수가 6~9에 몰리는 것을 막기 위해 앵커를 고정한다.

| 점수 | 의미 |
|---|---|
| 9~10 | 이 사용자에게 정확히 닿음, 폭넓고 구체적, "이건 내게 쓸 수 있다" |
| 7~8 | 대체로 좋으나 일부는 표면적/누구에게나 적용 가능 |
| 5~6 | 절반은 막연한 위로·바넘 |
| 3~4 | 대부분 일반론, 알맹이 적음 |
| 1~2 | 공허하거나 (위생 축이면) 신뢰 붕괴 |

**종합 산출**: 두 평균을 따로 낸다 — **효용 평균** = (축1+축2+축3)/3,
**경험 평균** = (축4+축5+축6)/3. **위생**은 **별도 표기**한다.
효용·경험·위생을 한 숫자로 뭉개지 않는 이유: "쓸모는 큰데 안 읽힌다",
"재밌는데 알맹이 없다", "효용·경험 다 좋은데 위생이 발목" 같은 **서로 다른
처방**이 필요한 상황을 한 평균이 가려버리기 때문이다.

### 2-A. 9점대 통과 게이트

"9점대 리포트"라고 부르려면 아래를 모두 만족해야 한다.

- 효용 평균 9.0 이상.
- 경험 평균 9.0 이상.
- 6개 축 중 8점 이하가 하나도 없어야 한다. 한 축이라도 8점 이하면 그 리포트는 "부분 우수"지 9점대가 아니다.
- 위생은 pass여야 한다. 치명 위반(미치환 변수, 한자·명리 용어 노출, 운명 단정·공포 조장, 가족 구성원 비난, JSON/구조 오류)이 있으면 평균이 높아도 9점대 탈락.
- 행동 처방은 최소 3개가 "발동 조건 / 실제 말·행동 / 시간 단위 / 완료 기준" 중 3개 이상을 갖춰야 한다.
- 장면 밀도: 핵심 섹션마다 최소 1개의 생활 장면이 있어야 하고, 전체 장면이 서로 다른 상황이어야 한다. 같은 비유·같은 처방의 반복으로 분량을 채우면 9점대 탈락.

## 3. 검증 방법론

### 반드시 지킬 것
- **중립 프롬프트**: 평가자에게 "이런 결함이 있을 것"이라는 가설을 미리
  주입하지 않는다(확증 편향 방지). 축 정의와 앵커만 준다.
- **객관 측정 병행**: LLM의 자기보고("4,000자 썼다")를 믿지 말고
  `tr -d ' \n\t' | wc -m`로 글자수, `grep`으로 누출/버그/비유 반복을 실측.
- **축 전담 분리**: 한 평가자가 5축을 뭉뚱그리지 말고, 축별 전담 에이전트가
  자기 축만 깊게 본다(이번 회차: 고민해소/자기이해/미래행동/위생 4명).

### 종류별 방법 차이
- **블라인드 역추론**(정답 안 주고 "이 사람 누구일까" 역추론 → 정답 대조)은
  **기질(TCI)에만** 적절하다. 입력 점수가 곧 정답이라 역추론 적중을 잴 수 있다.
  **사주에는 쓰지 말 것** — 사주의 정답은 TCI가 아니므로 역추론 대조가 무의미.
- **사주/융합/가족**은 효용 3축으로 평가하고, 자기이해 축에서만 "리포트가
  그리는 자기상"을 그려보되 TCI와 대조하지 않는다.

### 안티패턴 (하지 말 것) — 1차 검증의 실제 오류
1. ❌ 사주를 "TCI 페르소나(개척형 등)와 일치하는가"로 채점 → 사주를 부당하게 깎음.
2. ❌ 5축 동일가중 평균 → 핵심 실패(적중)가 문장·구체성 점수에 희석됨.
3. ❌ 점수 앵커 없이 채점 → 전부 6~8에 몰려 변별 안 됨.
4. ❌ 평가자에게 의심 지점을 미리 알려줌 → 그 결론으로 유도됨.
5. ❌ 사주↔기질이 다른 인물을 그린 것을 리포트 결함으로 감점 → 실은 통합 재료/제품 UX 이슈.

## 4. 검증 프로세스 (한 회차)

```
1) 렌더   npm run eval:render -- <persona>      # defaults.ts 기준, 비용 0
2) 생성   Claude 서브에이전트가 프롬프트→리포트   # out/<회차>/test_report_<종류>.txt
3) 측정   글자수·누출·반복 grep 실측             # 위생 1차 스크리닝
4) 평가   축별 전담 에이전트(중립·앵커)           # 4종 × 4축
5) 종합   <회차>/검증파일.md 작성                # 효용 평균 + 위생 별도 표기
```

리포트 종류 명명: `test_report_saju` / `test_report_tci` /
`test_report_saju_tci`(융합) / `test_report_family`.

## 5. 회차 폴더 규약

```
scripts/eval/out/
  1차 테스트/
    test_report_saju.txt  test_report_tci.txt
    test_report_saju_tci.txt  test_report_family.txt
    검증파일.md
  2차 테스트/   ← 프롬프트 개선 후 재생성·재검증, 같은 기준으로 점수 변화 비교
    ...
```

회차 간 비교가 목적이므로 **기준(이 문서)은 회차가 바뀌어도 고정**한다.
기준을 바꾸려면 이 문서를 먼저 고치고, 그 변경 이력을 아래 §7에 남긴다.

## 6. 한계 (정직 고지)
- 현재 생성·평가가 모두 Claude → 완전 독립 아님. 점수는 "프롬프트 구조의
  상한"이지 운영(Gemini) 품질이 아니다. 확정 개선본은 사용자가 실제 Gemini로
  1회 교차 검증할 것.
- "자기 이해" 축은 정답이 없어 "구체·일관·입체성"으로 대리 측정 → 진짜 사용자
  공감과는 차이가 있을 수 있음.
- 페르소나는 합성 표본. 다양한 케이스(평탄/시각모름/야자시/음력)는 회차를
  나눠 커버한다.

## 7. 기준 변경 이력
- **v0 (폐기)**: 사주를 "TCI 페르소나 적중"으로 평가 → 사주 부당 저평가(6.6, 꼴찌).
- **v1**: 사용자 효용 3축(고민해소·자기이해·미래행동) + 위생 별도.
  사주 효용 9.0 1위, 위생에서 감점 — "효용 충분, 위생 교정 필요"로 진단 정합.
- **v2**: 경험 3축 추가 — **새로움**(사주 경험자도 새롭게)·**재미**(글 자체가
  재밌게)·**쉬움**(어려운 용어 없이 술술). 효용/경험/위생을 각각 따로 표기.
  > ⚠️ v1로 평가한 1차 검증파일에는 경험 3축이 빠져 있다. 다음 회차부터 6축 적용.
- **v3 (현행)**: 9점대 통과 게이트 명시. 효용/경험 평균이 둘 다 9.0 이상이고, 8점 이하 축과
  위생 치명 위반이 없어야 "9점대"로 인정한다.

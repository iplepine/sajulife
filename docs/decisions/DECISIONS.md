## 2026-06-26 — 한국 출생 만세력에 기본 경도 보정 적용

**결정:** 출생 시각이 있는 한국 출생 만세력 계산은 역사 표준시/서머타임 보정 뒤, KST 기준 경도 동경 135도와 국내 평균 경도 동경 127.5도 차이를 반영해 추가로 30분을 뺀 계산 시각을 사용한다.

**이유:**
- 현재 프로필은 출생지를 받지 않으므로 지역별 경도 보정은 아직 적용할 수 없다.

- 기본 경도 보정을 계산층에 넣으면 리포트, 융합, 가족, 상담 컨텍스트가 같은 만세력 결과를 공유한다.

**한계:**
- 부산/서울 등 지역별 경도 차이와 균시차 기반 진태양시는 아직 적용하지 않는다.
- 해외 출생 타임존과 23시대 야자시/조자시 선택은 별도 정책 결정이 필요하다.

**결정:** 리포트 공유는 인증이 필요한 원본 데이터를 직접 노출하지 않고, `share:{token}`에 자기완결 스냅샷을 저장해 `/share/{token}`에서 공개 렌더한다.

**정책:**

- 재공유 시 같은 링크의 스냅샷을 최신 내용으로 갱신한다.

## 2026-06-20 — 상담은 리포트 원문이 아니라 요약 근거를 사용

**결정:** AI 상담은 저장된 리포트 원문 전체가 아니라 `user:{userId}:consult-basis`의 종류별 요약을 우선 사용한다. 요약이 없거나 낡았으면 상담 시 백필한다.

**이유:**
- 리포트 원문은 길어 상담 프롬프트 비용과 지연이 커진다.
- 사용자가 어떤 리포트를 근거로 쓸지 매번 고르게 하면 상담 진입이 느려진다.
- 존재하는 근거를 모두 합치는 기본값이 초기 UX에 적합하다.

**폴백:**
- 리포트가 하나도 없으면 상담 생성을 막고, 개인 사주 같은 리포트를 먼저 만들도록 안내한다.
- 리포트가 있는데 요약이 없거나 낡았으면 상담 시 백필한다.
- 요약 실패는 리포트 생성은 막지 않지만, 상담은 저장된 리포트 근거가 있어야 시작한다.

## 2026-05-29 — 이메일 회원가입/로그인 + 게스트→회원 연동

**결정:** Supabase Auth 위에 이메일/비밀번호 인증을 추가한다. 게스트(익명 세션)는 새 계정을 만드는 대신 기존 익명 계정에 이메일/비밀번호를 연동(`updateUser`)해 정식 회원으로 전환한다.

**이유:**
- 게스트 데이터는 `user.id`(auth.uid) 키로 KV에 저장된다.
- 익명 -> 정식 전환 시 `user.id`가 그대로 유지되어 별도 데이터 마이그레이션 없이 사주, 설문, 가족, 리포트가 보존된다.
- 새 계정을 만들고 데이터를 옮기는 방식은 충돌/유실 위험이 크다.

**구현:**
- `app/auth/login` — `signInWithPassword`.
- `app/auth/signup` — 세션이 익명이면 `updateUser({ email, password })`, 세션이 없으면 `signUp`.
- `app/auth/confirm/route.ts` — PKCE `code`와 OTP `token_hash` 모두 처리.
- `app/account` — 계정 상태, 게스트면 회원 전환 CTA, 로그아웃.
- 미들웨어는 `/auth/*`, `/share/*`, `/api/*`를 public으로 둔다.

**이메일 확인:** 프로토타입 단계에서는 OFF 운영 가능. 실제 결제 사용자 또는 외부 공개 시 커스텀 SMTP와 Confirm email ON을 다시 결정한다.

## 2026-05-26 — 기질 검사는 자체 프로토타입 문항으로 시작

**결정:** 정식 TCI 문항은 저작권/라이선스 이슈가 있으므로, 프로토타입은 자체 35문항 약식판으로 시작한다. 정식 140문항 슬롯은 별도로 두되 운영팀이 권리 있는 자료를 입력하기 전까지 실사용하지 않는다.

**이유:** thin-slice 목적은 정밀 심리검사가 아니라 리포트/상담 루프의 가치 검증이다.

**전환 트리거:** 기질 측정 정확도가 제품 신뢰/결제에 직접 영향을 준다는 근거가 나오면 정식 TCI 라이선스 또는 다른 검증된 도구를 검토한다.

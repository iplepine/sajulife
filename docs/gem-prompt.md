# Gemini Gems — 사주 리포트 테스트 가이드

새 프롬프트 구조를 운영에 올리기 전에 Gem에서 먼저 검증하는 워크플로우.
**우리 앱의 디버그 패널이 자동으로 만들어주는 완전 렌더된 프롬프트를 그대로 Gem에 붙여넣는 방식**으로 한다.
같은 프롬프트가 운영과 Gem 양쪽에서 동일하게 동작하므로 유지보수 비용이 없다.

---

## 1회만 — Gem 만들기

1. https://gemini.google.com 접속 (Gemini Advanced 구독 권장 — Pro 모델·thinking 사용 가능)
2. 좌측 메뉴 → **Gem 만들기**
3. **이름**: `사주 리포트 테스터`
4. **Instructions** 칸에 아래 ▼ 사이만 붙여넣기

```
당신은 사주명리 30년 경력의 한국 컨설턴트입니다. 다음 메시지로 주어지는 [사주 풀이 지침과 사용자 데이터]를 받으면, 그 지침에 명시된 형식·규칙·자체 검토 체크리스트를 모두 지켜 한국어 존댓말로 사주 리포트를 작성합니다. 지침에 없는 내용은 추가하지 않고, 지침에 명시된 마지막 자체 검토를 마음속으로 수행한 뒤 출력합니다.
```

5. 저장.

이게 끝. Instructions에는 어떤 사용자 데이터도, 출력 형식도 박지 않는다 — 그 모든 게 우리 앱에서 만들어지는 "완전 렌더된 프롬프트" 안에 이미 들어있다.

---

## 매번 — 한 사람 테스트하기

1. 우리 앱 ([/saju](http://localhost:3000/saju))에서 그 사람의 프로필 입력 → 리포트 생성
   - 무료 티어로 Flash로 한 번 생성 (≈₩30). 이 시점에 우리 시스템이 그 사람의 만세력·대운·오행 등을 다 계산해서 프롬프트를 렌더한다.
2. 리포트 페이지 하단에서 **디버그 보기** → **"Gem 프롬프트 복사"** 클릭
   - 클립보드에 그 사람용 완전 렌더 프롬프트가 들어감 (이름·생일·만 나이·4기둥·오행·음양한열·9대운까지 다 포함된 ~6,000자 텍스트)
3. Gem 새 채팅 → 첫 메시지로 **그대로 붙여넣기**
4. Gem이 ₩10k급 리포트를 작성 (Pro + thinking)

다른 사람을 테스트하려면 1번부터 다시 (그 사람 프로필로 바꿔서). 우리 앱이 매번 새 프롬프트를 렌더해주므로 수동 편집은 필요 없다.

---

## 평가 체크포인트

리포트를 받았을 때 다음을 점검:

- [ ] 첫 30초 안에 "오 이거 나야" 모먼트가 있는가
- [ ] **그림자 섹션**(1번의 그림자 3 / 7번의 자기변명 한 줄)이 진짜로 따끔한가
- [ ] 시간 표현이 "곧"이 아니라 "올해 12월" 같은 구체인가
- [ ] 4영역(일·돈 / 사랑·관계 / 몸 / 마음)이 다 별개의 결을 짚는가
- [ ] 90일 처방이 진짜 행동 가능한가
- [ ] 친구한테 보여줘도 되는 톤인가
- [ ] 사용자 이름이 본문에 5회 이상 자연스럽게 들어갔는가
- [ ] 총 단어 수가 3,000~3,500 사이인가
- [ ] 한자 사용이 사주 기둥/일간 표기 외에서 거의 없는가

---

## 프롬프트 자체를 손보고 싶을 때

운영 프롬프트는 [lib/prompts/defaults.ts](../lib/prompts/defaults.ts)의 `personal-saju.template`에 있다.
거기서 수정 → 앱 재시작 → 새로 생성 → "Gem 프롬프트 복사" → Gem에서 확인.

또는 admin UI (`/admin/prompts`)에서 직접 편집해도 됨 (DB 저장).

---

## 알려진 제약

- **무료 티어 API 키로는 Pro 모델을 호출할 수 없음**. 우리 앱에서 "다시 생성" 누르면 Flash로 떨어진다. 그래도 프롬프트는 동일하게 렌더되므로 Gem 측 결과는 정상.
- **Vercel 운영 환경 변수**: `GEMINI_MODEL`을 `gemini-2.5-pro`로 바꾸려면 결제 활성화된 API 키 필요.

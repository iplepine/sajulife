# Task: 출생시간 보정 정책 조사

상태: backlog
생성일: 2026-06-23
연결 Roadmap: `docs/work/roadmaps/active/2026-06-thin-slice-validation.md`

## 배경

개인 사주 입력에서 출생시간은 리포트 품질에 큰 영향을 주지만, 실제 보정 범위는 현재 사용하는 `lunar-javascript`의 계산 방식부터 확인해야 한다. 지금은 입력값을 그대로 계산에 사용하고, 보정 로직은 추가하지 않는다.

## 조사 범위

- `lunar-javascript`가 시주 경계, 자시/야자시, 음력 변환, 지역 표준시, 서머타임을 어떻게 처리하는지 확인한다.
- 한국 출생 기준과 해외 출생 기준을 분리할 필요가 있는지 판단한다.
- "시간 보정 필요" UI 문구가 필요한지, 필요하다면 어떤 정보(출생지, 실제 출생 시각, 표준시)를 추가로 받아야 하는지 정한다.

## 완료 기준

- 현재 계산 로직의 한계가 문서화되어 있다.
- 제품에서 지원할 보정 범위와 미지원 범위가 결정되어 있다.
- 구현이 필요하면 별도 active task로 승격한다.

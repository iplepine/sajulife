# 리포트 페르소나 반복과 한줄문구 말맛 정리

상태: Done  
기준 커밋: `c99af704589555561a85452a527492a67da7bae1`  
작성일: 2026-06-25  
완료일: 2026-06-25  
연결 roadmap: `docs/work/roadmaps/active/2026-06-thin-slice-validation.md`

## 배경

사주언니/기질오빠 브랜드 이미지를 리포트에 넣으면서 캐릭터 존재감은 생겼지만, 접히는 섹션마다 같은 얼굴이 반복되어 리포트 본문보다 페르소나 아이콘이 먼저 보이는 문제가 생겼다. 또한 섹션 한줄문구가 `~결`로 반복 종료되면서 실제 사람이 말하는 느낌보다 템플릿 문장처럼 읽힌다.

이번 작업은 브랜드 이미지를 없애는 것이 아니라, "필요한 곳에서만 강하게 쓰고 반복 구간에서는 조용히 빼는" 방향으로 정리한다.

## 목표

1. 리포트 상단에서는 사주언니/기질오빠 페르소나가 명확히 보인다.
2. 반복되는 접힘 섹션에서는 얼굴 이미지가 노이즈가 되지 않는다.
3. 섹션 한줄문구는 말풍선 UI가 아니라 리포트형 인용/강조 박스로 보인다.
4. 프롬프트가 `~결` 반복을 줄이고, 섹션별 말맛을 다르게 만든다.

## 작업 범위

### 1. 페르소나 이미지 사용 계층 정리

- 리포트 최상단 정체성/가이드 영역에는 페르소나 이미지를 유지한다.
- 개인 사주 첫 한줄 요약 카드에는 사주언니 얼굴 또는 얼굴 중심 크롭을 1회만 사용한다.
- 반복되는 접힘 섹션 헤더에서는 사주언니 얼굴을 제거한다.
- 섹션 종류 구분은 캐릭터 얼굴 대신 제목, 색상, 왼쪽 세로선, 작은 따옴표 장식으로 처리한다.
- 필요하면 `saju-unni-face.png`, `gijil-oppa-face.png`처럼 얼굴 중심 크롭 자산을 별도 생성한다.

예상 파일:

- `components/ReportView.tsx`
- `components/report/PersonalReportBody.tsx`
- `components/report/TciReportBody.tsx`
- `components/report/FusionReportBody.tsx`
- `app/globals.css`
- `public/brand-icons/*`

### 2. 한줄문구 강조 UI 정리

- 말풍선 꼬리는 사용하지 않는다.
- 문장 앞뒤에 실제 큰따옴표를 붙이지 않는다.
- 현재 박스형 강조 구조는 유지하되, 왼쪽 세로선과 작은 `“` 장식으로 "언니의 한마디" 느낌만 준다.
- 반복 섹션에서는 카드 안 카드처럼 보이지 않게 배경과 테두리 대비를 낮춘다.
- 모바일에서 한줄문구가 가로로 넘치지 않게 줄바꿈과 padding을 재점검한다.

예상 파일:

- `app/globals.css`
- `components/ReportView.tsx`

### 3. `~결` 반복 줄이기

프롬프트에 아래 제약을 추가한다.

- 리포트 전체에서 `결`은 브랜드 핵심어로 쓰되 남발하지 않는다.
- 같은 리포트의 섹션 한줄문구가 모두 `~결`로 끝나면 실패로 간주한다.
- 섹션 요약 끝맺음은 `흐름`, `패턴`, `리듬`, `주의점`, `관건`, `장면`, `필요해`, `쉬워`, `살아나` 등으로 분산한다.
- 정체성 대표 문장이나 리포트 제목에는 `결`을 쓸 수 있다.
- 하단 접힘 섹션 요약에서는 `결` 사용을 최대 1~2회로 제한한다.

예상 파일:

- `lib/prompts/defaults.ts`
- 필요 시 `scripts/eval/render.ts` 또는 리포트 품질 평가 스크립트

### 4. 리포트 샘플 검증

- 개인 사주 샘플 리포트를 생성하거나 기존 샘플을 렌더한다.
- 접힘 섹션 5개 이상에서 같은 얼굴이 반복되지 않는지 확인한다.
- 한줄문구의 끝맺음이 서로 다른지 확인한다.
- 데스크톱과 모바일에서 한줄문구 박스가 과하게 튀거나 잘리지 않는지 확인한다.

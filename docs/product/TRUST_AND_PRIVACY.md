# 신뢰와 개인정보 원칙

마지막 갱신일: 2026-06-20

## 다루는 데이터

현재 앱은 다음 개인 데이터를 다룬다.

- Supabase 사용자 식별자와 이메일.
- 이름 또는 별명.
- 생년월일, 출생 시각, 양/음력, 성별.
- 사용자가 남긴 메모.
- TCI 응답과 점수.
- 가족 구성원의 이름, 관계, 생년월일, 출생 시각, 양/음력, 성별.
- 리포트 본문, 상담 질문/답변, 코칭 액션.
- 공개 공유 스냅샷.

## 저장 위치

- 인증 세션과 이메일 계정은 Supabase Auth가 관리한다.
- 제품 데이터는 Upstash Redis KV에 `user:{userId}:...` 네임스페이스로 저장한다.
- 공개 공유 링크는 `share:{token}` 키에 자기완결 스냅샷으로 저장한다.
- 프롬프트 override는 전역 `prompts` 키에 저장한다.

## LLM 전송 범위

Gemini 호출에는 기능별로 다음 데이터가 포함된다.

- 개인 사주 리포트: 프로필, 계산된 사주표, 대운/오행/음양한열, 메모.
- 기질 리포트: 이름, 성별, TCI 점수.
- 융합 리포트: 프로필, 계산된 사주표, TCI 점수.
- 가족 리포트: 본인 프로필/사주, 가족 구성원 프로필/사주.
- 상담: 질문, 저장된 리포트 요약. 리포트가 없으면 상담을 생성하지 않는다.
- 상담 근거 요약: 리포트 본문 일부 또는 전체.

현재 Explore 단계에서는 Google AI Studio/Gemini 설정에 따라 입력 데이터가 모델 개선에 사용될 수 있는지 별도 확인이 필요하다. 외부 베타 또는 결제 사용자 전환 전에는 paid tier, 데이터 사용 조건, 개인정보 처리방침을 확정해야 한다.

## 공개 공유 정책

- `/share/{token}` 링크는 인증 없이 열린다.
- 링크를 아는 사람은 누구나 해당 리포트 스냅샷을 볼 수 있다.
- 같은 사용자와 리포트 종류는 같은 토큰을 재사용하고, 재공유 시 최신 스냅샷으로 갱신한다.
- 현재 revoke/delete UI는 없다.
- 가족 리포트 공유는 타인의 출생 정보와 관계 해석이 포함될 수 있어 추가 경고가 필요하다.

## 해석 신뢰 원칙

- 만세력 계산은 LLM이 아니라 `lunar-javascript` 결과를 사용한다.
- LLM에는 계산된 값을 인용하게 하고, 재계산하지 못하게 프롬프트에서 제한한다.
- 사주, 기질, 가족 해석은 자기 이해/상담 콘텐츠이지 의료, 법률, 투자, 안전 판단을 대체하지 않는다.
- 반말 팩폭 톤을 쓰더라도 모욕, 공포 조장, 가족 구성원 비난은 금지한다.

## 제품화 전 필수 작업

- 개인정보 처리방침과 이용약관 작성.
- Gemini paid tier와 데이터 사용 조건 확정.
- 사용자 데이터 삭제 요청 처리 경로.
- 공개 공유 링크 revoke 또는 만료 정책.
- 가족 구성원 데이터 입력/공유에 대한 명시 고지.
- 결제 전환 시 이메일 확인, 환불, 고객지원 정책.

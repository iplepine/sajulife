# MVP 범위

마지막 갱신일: 2026-06-20

## MVP 목표

현재 MVP는 "사주 기반 생애 문제 컨설팅 루프"가 실제로 작동하는지 검증한다. 만세력 정밀 기능, 결제, 자동 리마인더보다 사용자가 리포트를 읽고 상담/액션으로 넘어가는지가 우선이다.

## 포함 범위

- 익명 게스트 시작과 이메일 계정 전환.
- 사주 프로필 저장과 수정.
- 약식 TCI 35문항 저장/이어풀기, 점수화, 기질 리포트 생성.
- 정식 TCI 140문항 슬롯. 운영팀이 라이선스 문항을 채워야 실제 사용 가능.
- 개인 사주 리포트 생성/저장/재생성.
- 사주 x 기질 융합 리포트 생성/저장/재생성.
- 가족 구성원 추가/수정/삭제, 가족 리포트 생성/저장/재생성.
- AI 상담 질문 입력, 저장된 리포트 요약 기반 답변, 상담 히스토리.
- 코칭 액션 등록, 직접 추가, 완료 토글, 삭제.
- 리포트 공개 공유 링크 생성과 OG 이미지.
- 프롬프트 기본값, KV override, 관리자 수정/리셋 API.
- 오프라인 프롬프트 렌더/eval 하니스.

## 제외 범위

- 결제, 구독, 유료 리포트 권한.
- 자동 이메일/푸시 리마인더.
- 리포트 히스토리 보관. 현재 리포트 4종은 종류별 최신본만 저장하고 재생성 시 덮어쓴다.
- 공유 링크 폐기/revoke UI. 같은 사용자/리포트 종류는 같은 토큰을 재사용하고 최신 스냅샷으로 갱신한다.
- 데이터 내보내기/삭제 요청 UX.
- 임상 TCI 정식 라이선스 검증. 정식 140문항은 자료 입력과 권리 확인 전까지 실사용 금지.
- 의료, 법률, 투자 의사결정 조언.
- 자동화된 E2E/단위 테스트. 현재는 타입체크/빌드/eval 하니스와 수동 QA 중심이다.

## 핵심 사용자 흐름

1. 랜딩에서 게스트로 시작한다.
2. 사주 프로필을 입력한다.
3. 대시보드에서 개인 사주 또는 기질 검사를 시작한다.
4. 리포트를 생성하고 읽는다.
5. 융합 리포트 또는 AI 상담으로 넘어간다.
6. 리포트/상담의 액션 후보를 코칭 플랜에 등록한다.
7. 코칭 플랜에서 실행 여부를 체크한다.
8. 필요하면 리포트를 공유하거나 이메일 회원으로 전환한다.

## 완료 기준

MVP가 검증됐다고 볼 수 있는 최소 조건:

- 베타 사용자 5명 이상이 프로필 입력부터 리포트 생성까지 완료한다.
- 그중 3명 이상이 AI 상담에 실제 고민을 입력한다.
- 그중 2명 이상이 코칭 액션을 등록하거나 직접 추가한다.
- 인터뷰에서 "리포트만 읽고 끝"이 아니라 "다음 고민에도 써볼 수 있다"는 신호가 나온다.

## 현재 구현상 제약

- AI 모델 기본값은 `gemini-2.5-pro`이며, 결제 활성화 API 키가 필요하다. 비용 제약이 있으면 `.env.local`의 `GEMINI_MODEL`을 Flash 계열로 낮춘다.
- 상담 근거 요약은 기본 `gemini-2.5-flash`를 사용한다. 요약 실패는 리포트/상담 생성을 막지 않지만 상담 근거 품질이 낮아질 수 있다.
- 리포트 생성 API는 비용이 발생하므로 자동 테스트에서 직접 호출하지 않는다.
- 개인정보 처리방침/약관은 아직 제품화 전 문서로 정리되지 않았다.
- 프롬프트 debug UI는 관리자 편집용이며, 일반 사용자에게 노출되는 제품 플로우가 아니다.

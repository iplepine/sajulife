# 출시 준비 상태

마지막 갱신일: 2026-06-20

## 현재 판단

외부 유료 출시 준비는 아직 부족하다. 내부/소수 베타 테스트는 가능하되 개인정보, 공유, 비용, 약관, 모델 데이터 처리 조건을 명시적으로 관리해야 한다.

## 준비 상태 체크리스트

| 영역 | 상태 | 메모 |
|---|---|---|
| 로컬 실행 | 부분 준비 | `.env.example` 존재, Vercel env pull 흐름 있음 |
| 배포 설정 | 부분 준비 | `vercel.json`과 Next 설정 존재 |
| 인증 | 부분 준비 | Supabase 익명/이메일 구현, 이메일 확인 정책은 운영 결정 필요 |
| 저장소 | 부분 준비 | Upstash KV 사용, 삭제/백업/마이그레이션 정책 없음 |
| AI 모델 | 부분 준비 | Gemini Pro 기본, 비용/데이터 사용 정책 확인 필요 |
| 개인정보 고지 | 미준비 | 처리방침/약관 없음 |
| 결제 | 미구현 | 수익 모델 가설만 있음 |
| 테스트 | 미준비 | 자동 unit/E2E 없음, typecheck/build/eval 중심 |
| 공유 | 부분 준비 | 공개 링크 구현, revoke/민감 정보 경고 부족 |
| 고객지원 | 미준비 | 문의/삭제 요청 경로 없음 |
| 분석 | 부분 준비 | Vercel Analytics 전환 이벤트 일부 |

## 베타 공개 전 필수

- `npm run typecheck` 통과.
- `npm run build` 통과.
- 최소 1명 프로필 입력 -> 개인 리포트 -> 상담 -> 액션 등록 수동 QA.
- 공유 링크가 비로그인 브라우저에서 열리는지 확인.
- 개인정보/LLM 데이터 사용 안내를 랜딩 또는 온보딩에 최소 고지.
- Gemini API 키가 paid tier인지, 입력 데이터 사용 조건이 무엇인지 확인.
- Supabase Anonymous Sign-ins와 Email provider 설정 확인.
- 운영 도메인의 Supabase Redirect URL 등록.

## 유료 출시 전 필수

- 개인정보 처리방침/이용약관/환불 정책.
- 결제 플로우와 권한 모델.
- 사용자 데이터 삭제 요청 처리.
- 공개 공유 링크 revoke 또는 만료 정책.
- 가족 구성원 데이터 입력 동의/고지 강화.
- 최소 핵심 API/도메인 로직 테스트.
- 비용 상한, rate limit, 에러 모니터링.
